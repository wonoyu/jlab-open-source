# API Integration Tutorial

Welcome to the API Integration tutorial for wdev_cli. In this guide, you'll learn how to set up API services, define endpoints, generate API clients, and integrate services into your Flutter features.

:::info
This tutorial assumes you have a Flutter monorepo initialized with `wdev init`. If you haven't set up your project yet, please complete the [Getting Started Tutorial](../tutorial.md) first.
:::

## What You'll Learn

By the end of this tutorial, you will be able to:

- Add new API services with per-flavor configuration
- Define API endpoints using JSON files
- Generate Retrofit API clients automatically
- Inject services into features with complete service layers

## Prerequisites

Before starting this tutorial, ensure you have:

1. **A Flutter monorepo initialized** with `wdev init`
2. **At least one flavor configured** in your project
3. **Flutter SDK installed** (version 3.0 or higher)
4. **Dart SDK installed** (version 3.0 or higher)

:::tip
To verify your setup, run `wdev --version` to confirm wdev_cli is installed correctly.
:::

## Chapter 1: Adding Your First API Service

In this chapter, you'll create a new API service for managing users.

### Step 1.1: Initialize the Service

Run the following command to add a new API service:

```bash
wdev api add --name users
```

The command will prompt you for the base URL for each flavor in your project:

```text
Adding new API service...

Detected flavors: dev, staging, prod

? Service name (e.g., users): users
? URL for dev (e.g., https://users-dev.example.com): https://users-dev.example.com
? URL for staging (e.g., https://users-staging.example.com): https://users-staging.example.com
? URL for prod (e.g., https://users-prod.example.com): https://users-prod.example.com
```

### Step 1.2: Verify Service Creation

After completing the prompts, the command creates:

```
packages/api/lib/openapi/users/
├── users.yaml          # OpenAPI specification
├── config.dart         # Per-flavor configuration
└── endpoints/          # Endpoint definitions
```

### Step 1.3: Non-Interactive Mode

For CI/CD pipelines or scripts, provide all options upfront:

```bash
wdev api add --name users \
  --output packages/api/lib/openapi \
  --no-examples \
  --force
```

## Chapter 2: Defining API Endpoints

Now that you have a service, let's add some endpoints.

### Step 2.1: Create Endpoint Definitions

Create a JSON file for each endpoint. For example, `packages/api/lib/openapi/users/get_users.json`:

```json
{
  "method": "GET",
  "path": "/users",
  "operationId": "listUsers",
  "summary": "List all users",
  "description": "Returns a paginated list of users",
  "parameters": [
    {
      "name": "page",
      "in": "query",
      "schema": { "type": "integer", "default": 1 }
    },
    {
      "name": "limit",
      "in": "query",
      "schema": { "type": "integer", "default": 20 }
    }
  ],
  "responses": {
    "200": {
      "description": "Successful response",
      "content": {
        "application/json": {
          "schema": {
            "type": "object",
            "properties": {
              "data": {
                "type": "array",
                "items": { "$ref": "#/components/schemas/User" }
              },
              "meta": {
                "type": "object",
                "properties": {
                  "page": { "type": "integer" },
                  "total": { "type": "integer" }
                }
              }
            }
          }
        }
      }
    }
  }
}
```

### Step 2.2: Add Endpoints to Service

Use the `add-endpoint` command to process your JSON files:

```bash
wdev api add-endpoint --source packages/api/lib/openapi/users
```

### Step 2.3: Watch Mode for Development

During development, use watch mode to regenerate automatically:

```bash
wdev api add-endpoint --source packages/api/lib/openapi/users --watch
```

## Chapter 3: Generating API Clients

Now generate Retrofit clients from your OpenAPI specifications.

### Step 3.1: Run the Generator

Execute the generate command to create Dart API client code:

```bash
wdev api generate
```

This command:

1. Reads all OpenAPI specifications from `packages/api/lib/openapi/`
2. Generates Retrofit client classes
3. Creates model classes for all schemas
4. Outputs to `packages/api/lib/src/`

### Step 3.2: Generated Files

After generation, you'll have:

```
packages/api/lib/src/users/
├── users_api.dart         # Retrofit API service
├── models/
│   ├── user.dart
│   └── user.freezed.dart
└── users_api.g.dart       # Generated by build_runner
```

### Step 3.3: Using the Generated API

Import and use the generated API client:

```dart
import 'package:my_project_api/users_api.dart';

final usersApi = UsersApi.create();

final response = await usersApi.listUsers(page: 1, limit: 20);
final users = response.data;
```

## Chapter 4: Injecting Services into Features

The final step is to inject your API service into a feature with a complete service layer.

### Step 4.1: Basic Service Injection

Inject a service into a feature page:

```bash
wdev api integrate \
  --service users \
  --page home \
  --feature dashboard \
  --endpoint getUserById
```

### Step 4.2: What Gets Generated

The integrate command creates a complete service layer:

```
lib/src/features/dashboard/
├── home/
│   ├── home_page.dart
│   ├── home_viewmodel.dart
│   └── widgets/
├── services/
│   └── users_service.dart
├── repositories/
│   └── users_repository.dart
├── models/
│   └── user.dart
└── providers/
    └── users_provider.dart
```

### Step 4.3: Using the Service in Your UI

After injection, use the service in your feature:

```dart
import 'package:my_project/features/dashboard/providers/users_provider.dart';

class HomePage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final userAsync = ref.watch(usersProvider);

    return userAsync.when(
      data: (user) => Text('Hello, ${user.name}'),
      loading: () => CircularProgressIndicator(),
      error: (err, stack) => Text('Error: $err'),
    );
  }
}
```

## Chapter 5: Advanced Integration Options

### Step 5.1: Adding Caching

Enable local caching with Drift:

```bash
wdev api integrate \
  --service users \
  --page home \
  --feature dashboard \
  --endpoint getUserById \
  --cache \
  --cache-keys id
```

### Step 5.2: Offline-First with PowerSync

Enable PowerSync for offline-first functionality:

```bash
wdev api integrate \
  --service users \
  --page home \
  --feature dashboard \
  --endpoint getUserById \
  --powersync
```

### Step 5.3: Generating All Endpoints

Generate all endpoints from a service:

```bash
wdev api integrate \
  --service users \
  --page home \
  --feature dashboard \
  --all-methods
```
